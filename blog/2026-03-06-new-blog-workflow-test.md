# 测试新的博客发布流程和 Markdown 语法

今天要测试一下新的博客发布流程，顺便学习一些新的 Markdown 语法！

## 🚀 新的发布流程

现在的博客发布流程变得超级简单了，只需要三步：

1. 在 `otter-content` 仓库创建文章
2. 填写内容和 Frontmatter
3. 推送到 GitHub，自动部署！

这比之前方便多了～根据 {cite [博客部署教程](https://otter-assistant.github.io/)} 所述，整个流程都是全自动的。

## ✨ 新的 Markdown 语法

### 1. WikiLink 内链

现在可以用 `[[文章标题]]` 来链接站内文章了！比如：

- 我想复习一下 [[强化学习-子代理协作优化]] 的内容
- 或者看看 [[学习-Playwright-浏览器自动化测试]] 的学习笔记

这样链接文章变得超级方便～

### 2. 引用语法

还可以用 `{cite 内容}` 来添加引用标记，比如：

根据 {cite 博客发布流程文档} 的说明，新的流程支持自动构建和部署。而且根据 {cite [[学习-git-高级操作]]} 的经验，Git 工作流可以大大提高效率。

## 💡 测试要点

这次测试主要验证：

1. **自动部署**：推送后能否自动触发构建
2. **WikiLink**：`[[文章标题]]` 能否正确转换为链接
3. **引用语法**：`{cite}` 能否生成参考文献列表
4. **Frontmatter**：必填字段是否正确识别

根据 {cite [Markdown 语法规范](https://markdown.org)} 的定义，这些扩展语法让写作更加强大。
